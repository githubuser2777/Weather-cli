# Requirements

## User Stories

1. [x] As a user, I want to type `weather-cli` and get the weather for my current location without passing any arguments.
2. [x] As a user, I want to see a clean, color-coded display so it's easy to read at a glance.
3. [x] As a user, I want to query a specific city with `--city "London"`.
4. [x] As a user, I want the CLI to run smoothly on my Windows PowerShell and my Linux terminal.

## Acceptance Criteria

### Core Interactions
- [x] `weather-cli --city "Paris"` fetches weather for Paris.
- [x] `weather-cli` auto-detects IP location and fetches weather.
- [x] `weather-cli --help` prints usage instructions.

### Presentation
- [x] Output includes Temperature, Humidity, and Conditions.
- [x] Colors are used to represent conditions (e.g., Red for hot, Blue for cold).
- [x] Windows CMD/PowerShell display colors correctly.

## Non-Functional Requirements

- [x] **Performance:** Command must execute and display within 2 seconds.
- [x] **Dependencies:** Standard library only (`net/http`, `encoding/json`, `flag`).

## Definition of Done

- [x] All code is in `cmd/` and `internal/`.
- [x] Tests are written for `internal/weather` and `internal/location`.
- [x] Build succeeds for both `windows/amd64` and `linux/amd64`.
